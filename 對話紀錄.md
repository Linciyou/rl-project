# 從 0 到 1 開發 UPW 強化學習再生排程系統：對話紀錄

> 本文件是本次在這台電腦上開發 `化工廠.py` 專案的整理版對話紀錄，不是逐字稿。內容描述需求、OpenSpec 規格、系統設計、程式建構、README 撰寫與完成狀態。

## 1. 初始需求

**使用者：**

請從 0 開始建構一個 UPW 強化學習再生排程系統，並撰寫 README。

**Codex：**

先建立專案目標：在這台 Windows 電腦上開發一個可維護的本機 Python 系統，主入口檔名為 `化工廠.py`，核心邏輯放在 `upw_rl_system/` package 中。

系統需要支援：

- 從 CSV 讀取 UPW 製程歷史資料
- 驗證必要欄位
- 進行資料前處理與標準化
- 建立 PINN/LSTM 數位雙生模型
- 建立 Gymnasium 模擬環境
- 使用 Stable-Baselines3 PPO 訓練再生策略
- 產生模擬的混床再生週期排程
- 撰寫 README 說明安裝、資料格式、使用方式與輸出

## 5. Design 階段

**Codex 建立：**

`openspec/changes/modularize-rl-system/design.md`

設計決策：

- `化工廠.py` 作為使用者主要執行入口
- `upw_rl_system` 作為可維護的 Python package
- 使用 `python 化工廠.py ...` 與 `python -m upw_rl_system ...` 兩種入口
- 使用 `argparse` 建立 CLI
- 輸出 artifacts 集中放在 `outputs/`
- dependency 安裝資訊放在 `requirements.txt` 與 README

規劃模組：

```text
upw_rl_system/

  data.py         # CSV 驗證、篩選、標準化、transition 建立

  environment.py  # Gymnasium 模擬環境
  pipeline.py     # 端到端訓練與排程流程
  cli.py          # 命令列介面
  __main__.py     # python -m upw_rl_system 入口
```

## 10. `data.py`

**Codex 實作：**

建立資料處理流程：

- 讀取 CSV
- 驗證必要欄位
- 依指定 `arm` 篩選資料
- 處理缺值
- 將 `MB_regen` 轉為二元 action
- 使用 `StandardScaler` 標準化製程特徵
- 建立 state transition 訓練資料

## 11. `model.py`

**Codex 實作：**

建立 `UPWPinnLSTM` 數位雙生模型。

模型輸入：

- 當前標準化 state
- binary regeneration action

模型輸出：

- 下一週標準化 state
- state delta

訓練函式加入：

- data loss
- no-maintenance TOC physics penalty
- maintenance TOC drop physics penalty

## 12. `environment.py`

**Codex 實作：**

建立 `PINNSimulatedUPWEnv`，符合 Gymnasium 環境格式。

環境設定：

- action space：`Discrete(2)`
- observation space：製程特徵向量
- reset 時設定 TOC baseline
- step 時用 PINN/LSTM 推進下一週狀態
- 根據 TOC、再生成本與終止條件計算 reward

## 13. `pipeline.py`

**Codex 實作：**

建立端到端流程：

1. 設定 random seed
2. 載入 CSV 並前處理
3. 訓練 PINN/LSTM digital twin
4. 建立 UPW 模擬環境
5. 訓練 PPO policy
6. deterministic rollout
7. 印出每週排程
8. 儲存 artifacts

預設 artifacts：

- `digital_twin.pt`
- `preprocessing.pkl`
- `ppo_policy.zip`
- `schedule.csv`
